ES2015 Classes

  • Classes should be named using PascalCase
  • Class file names should match the name of the class, e.g. MyNewClass.js
  • There should only be 1 class per file
  • Classes should contain a comment block above their definition which provides a short summary
  • static methods should be placed at the top of the class (below the constructor) in alphabetical order

ES2015 Classes

  • Private functions and variables should be prefixed with an underscore ( _ )
  • All methods should have a comment block above their definition which describes their function. This should make use of jsdoc annotations to inidicate parameters and their type, related functions, author, return value(s), access level (public, private), etc.
